## 1. Alt у изображений

**Проверка по коду (25.02.2026):**

- Все ключевые изображения имеют осмысленный `alt`:
  - **Логотипы** (`layouts/default.vue`): `alt="Сетки 21"`.
  - **Главная** (`index.vue`): HeroImage с `alt="Рамочная москитная сетка на пластиковом окне в Чебоксарах и Новочебоксарске"`.
  - **Категории** (антимошка, антикошка, ультравью, антипыль, вставные, ремонт): HeroImage с уникальным alt (тип сетки + гео или описание замеров).
  - **HeroSection**: использует `imageAlt || title` — при вызове компонента нужно передавать `imageAlt`, иначе подставится заголовок.
  - **Админка/дилер** (логотип дилера): `alt="Логотип дилера"`.

**Что делать дальше:**

- [ ] При добавлении новых `<img>` или компонентов с картинками всегда задавать уникальный `alt` (не пустой, не имя файла).
- [ ] Если где-то используется HeroSection без `imageAlt` — передать осмысленный `imageAlt` для каждой страницы.

## 2. robots.txt для Yandex

**Текущее состояние:**

- Для `User-agent: Yandex` заданы:
  - `Crawl-delay: 2`
  - `Clean-param: utm_source&utm_medium&utm_campaign&utm_content&utm_term`
- Общие правила (User-agent: *) применяются к Yandex в части Allow/Disallow (в т.ч. Allow: /images/, /upload/, Disallow: /admin/ и т.д.).

**Что делать дальше:**

- [ ] Точечные правки не обязательны — конфиг для Yandex уже адекватный.
- [ ] При желании можно явно добавить для Yandex блок с `Allow: /` и повтором Allow для /images/ и /upload/ (для ясности), но текущего достаточно.

## 3. Цели в Яндекс.Метрике

**Что уже отправляется из кода:**

- **ORDER_SUBMITTED** — при отправке заявки из калькулятора (`Calculator.vue`). Параметр: `{ price: store.totalPrice }`.
- **CALL_CLICK** — при клике по ссылке/кнопке «Позвонить» в шапке и на странице 404 (`layouts/default.vue`, `error.vue`).

Счётчики в коде: `48599813`, `63315469` (основной + Яндекс.Бизнес). Инициализация и отправка целей — `plugins/metrika.client.ts`, глобальная функция `window.reachMetrikaGoal(goalName, params)`.

**Что сделать в интерфейсе Метрики:**

- [ ] Зайти в **Настройки → Цели** для счётчика 48599813 (и при необходимости для 63315469).
- [ ] Создать цель с идентификатором **ORDER_SUBMITTED** (тип «JavaScript-событие», идентификатор цели = `ORDER_SUBMITTED`).
- [ ] Создать цель с идентификатором **CALL_CLICK** (тип «JavaScript-событие», идентификатор = `CALL_CLICK`).
- [ ] В разделе **Отчёты → Стандартные отчёты → Конверсии** или **Отчёты → Посещаемость** проверить срабатывание целей в режиме «Реальное время» (оставить вкладку открытой, отправить заявку и нажать «Позвонить»).

**Опционально (на будущее):**

- Цели «Просмотр каталога», «Добавление в корзину» и т.п. — добавить в код вызовы `reachMetrikaGoal('GOAL_NAME')` и затем создать соответствующие цели в Метрике.
